## 2. Пересекающиеся требования

Текстовая диаграмма Венна:

- Заявитель ∩ Координатор: FR-03 «Пользователь может открыть карточку выбранной заявки».
- Координатор ∩ Администратор: FR-02 «Пользователь может просматривать список заявок», FR-19 «Система должна вести историю изменения статуса заявки».
- Координатор ∩ Волонтер: FR-09 «Пользователь может отметить заявку как выполненную» с разными правами доступа в реальной версии.
- Все роли: просмотр актуального состояния заявки через список или карточку.

Если действие одинаковое, оно оформлено одним требованием с перечнем ролей. Если различаются права или результат, требование разделено: например, координатор назначает волонтера, а волонтер только завершает назначенную работу.
