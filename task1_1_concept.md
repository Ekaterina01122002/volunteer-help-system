# task1_1. Концепция программного проекта

Проект: ПС «Организация волонтерской помощи»

Источник задания: [task1_1.md](https://github.com/olgmina/SWEngineering-technics.github.io/blob/gh-pages/0_Requierements/task1_1.md)

## 1. Описание проблемы

**Проблема.** Небольшие волонтерские центры часто ведут заявки на помощь в мессенджерах, таблицах или бумажных списках. Из-за этого координатору сложно видеть актуальный статус каждой заявки, быстро назначать волонтеров, отслеживать просрочки и заранее учитывать погодные риски для уличной помощи.

**Воздействует на:**

- заявителей, которым нужна адресная помощь;
- волонтеров, которым нужны понятные назначения и актуальная информация;
- координаторов, отвечающих за распределение заявок;
- операторов/администраторов, поддерживающих справочники и корректность данных.

**Результатом проблемы является:**

- потеря или дублирование заявок;
- задержка помощи из-за несвоевременного назначения волонтера;
- отсутствие единой истории статусов;
- повышенная нагрузка на координатора;
- риск выполнения уличной заявки без учета погодных условий.

**Выигрыш от программного продукта:**

- единый список заявок и волонтеров;
- автоматическая смена статусов по бизнес-правилам;
- контроль заявок старше 3 дней без назначенного волонтера;
- формирование сообщения координатору при просрочке;
- погодное предупреждение для уличных заявок;
- локальное хранение данных и демонстрационные данные для учебного запуска.

## 2. Назначение программного средства

ПС предназначено для учета заявок на волонтерскую помощь, назначения волонтеров, контроля статусов, проверки просрочек и формирования уведомлений координатору.

**Кто применяет систему:**

- волонтерский центр;
- учебная команда разработки;
- координатор благотворительной или социальной организации;
- оператор, который принимает заявки по телефону.

**Входные данные:**

- данные заявителя: ФИО, телефон;
- данные заявки: тип помощи, тип места, адрес, дата и время, описание;
- данные волонтера: ФИО, телефон, навыки, доступность;
- настройки координатора и погодного API;
- погодные данные из OpenWeatherMap API или учебной заглушки.

**Выходные данные:**

- список заявок;
- карточка заявки;
- история статусов;
- список волонтеров;
- назначение волонтера;
- погодное предупреждение;
- сообщение координатору о просрочке;
- статистика по статусам.

**Сторонние системы и сервисы:**

- OpenWeatherMap API для проверки погодных условий;
- локальное JSON-хранилище `data/app-data.json`;
- Maven Central для зависимостей проекта.

## 5. Описание продукта по шаблону Moore

**Для** координаторов небольших волонтерских центров, **которые** принимают адресные заявки на помощь и назначают волонтеров вручную, **этот** программный продукт «Организация волонтерской помощи» **является** desktop-системой учета заявок, **которая** помогает контролировать статусы, просрочки и погодные риски. **В отличие от** таблиц, мессенджеров и универсальных онлайн-платформ, **наш продукт** ориентирован на учебную локальную работу координатора и содержит готовые правила жизненного цикла заявки, уведомление о просрочке и погодную проверку.
